Data Binding

Data Binding in AngularJS is the automatic synchronization of data between model and the view components. The view is a projection of the model at all times. When the model changes, the view reflects the change, and vice versa. There is huge different between Data Binding in Classiscal Template Systems and Angular Templates.

Classical Template

Most templating systems bind data in only one direction: they merge template and model components together into a view. After the merge occurs, changes to the model or related sections of the view are NOT automatically reflected in the view. Worse, any changes that the user makes to the view are not reflected in the model. This means that the developer has to write code that constantly syncs the view with the model and the model with the view.

Data Binding in Classical Template

AngularJS Template

AngularJS templates work differently. First the template (which is the uncompiled HTML along with any additional markup or directives) is compiled on the browser. The compilation step produces a live view. Any changes to the view are immediately reflected in the model, and any changes in the model are propagated to the view. Because of this kind of binding, testing an angular app is very easy.

Data Binding in Angular Template

We can do data binding in angular various ways. One way is to use ng-bind. For example:
<p> ng-bind="firstname"></p>
This will bind innerHTML of the element to the specified model property. Another way is to use double braces {{}}. For example:
<p>First name: {{firstname}}</p>
This will display content from the model.

Two way Binding

The ng-model directive also provides a two-way binding between the model and the view. I personlly find two way binding in Angular very cool. For example:
<input ng-model="firstname">
